Carrot2 4.0.x

Release history for Carrot2 4.0.x and bugfix releases.

Version 4.1.0 [unreleased]

This release adds ephemeral per-request dictionaries and introduces minor adjustments to Java and REST API formats.

  • GH-43: [DCS, Java, API CHANGE] Lingo algorithm's filter attributes have been changed from Booleans to proper objects with a dedicated enabled attribute. Unless you used these attributes explicitly, no action is needed.

  • GH-44: [DCS, Java, API CHANGE] Ephemeral (per-request) label and word filtering support has been added. This feature allows passing per-request word and cluster label filters, without reloading the default LanguageData resources. See ephemeral dictionary section in the Java API and REST API sections for more information.

  • GH-45: [Java, API CHANGE] The LexicalData interface (LanguageData component) has been split into two independent components: StopwordFilter and LabelFilter. The default implementations and abstract classes have been changed accordingly.

  • GH-38: [DCS, API CHANGE] Expose the language and algorithm of all resolved template in /list. The response format of the endpoint has changed. Previously, the templates element was a list of template names, now it will contain an object with template names as keys and template content as values, for example: 

    ...
  "templates" : {
    "template1" : {
      "language" : "English",
      "algorithm" : "algorithm1"
    },
    "template2" : {
      "algorithm" : "algorithm1"
    }
  }

  • GH-37: [build] Only recompile code under IntelliJ IDEA (improve dev experience).

  • GH-35: [DCS] Added request processing/ service information to clustering response. This information is optional and is returned when serviceInfo HTTP parameter is enabled on a clustering request.

  • GH-34, CARROT-1280: [DCS] Clustering of documents containing multi-value fields (array of strings) fails with an exception.

  • GH-33: [DCS] Upgrade Jackson dependency to 2.11.3.
