Algorithms

This section lists the algorithms available in Carrot2

Each clustering algorithm tackles the problem of identifying groups of related documents in a slightly different way and comes with different performance characteristics (see tuning performance for deeper insight into each algorithm's performance profile).

Carrot2 offers several clustering algorithms to choose from: Lingo, STC and a generic bisecting k-means clustering. In addition to these open source implementations, a commercial algorithm Lingo3G plugs seamlessly into the same open source API. Switching between any of them is trivial.

The choice of an algorithm will affect the clustering output but no algorithm is "the best" (refer to this section for details). Experiment with all of them, tweak parameters, finally select what fits your particular needs and expectations. Lingo algorithm

The Lingo algorithm employs term-document matrix dimensionality reduction techniques to figure out the structure of topics present in the input. The algorithm is reasonably fast and nicely separates diverse topics into separate groups. This will typically be the default algorithm you may want to use.

STC algorithm

The key data structure in the STC algorithm is the Suffix Tree or rather a Generalized Suffix Tree (GST) built for all input documents. The algorithm traverses the GST to identify words and phrases that occurred frequently in the input documents and merges sub-groups of documents with high overlap. The algorithm is very fast, even for large sets of documents, but suffers from lack of cluster diversity.

Bisecting k-means

Bisecting k-means is an implementation of a generic cluster-analysis technique (contrary to Lingo and STC which are text-specific). The algorithm attempts to recursively split the documents into smaller and smaller groups until the splits become unfeasible. The disadvantage of this algorithm is that the labels it creates are often subpar compared to STC or Lingo (they consist of single terms rather than phrases).

Lingo3G algorithm

Lingo3G is a commercial successor to the Lingo algorithm. The algorithm offers faster performance, lower memory consumption and many more tuning options compared to Lingo. Please refer to Carrot Search Lingo3G web site for details.
