REST API Basics

This article will walk you through the methods offered by the Carrot2 HTTP/REST service.

Carrot2 comes with a component that offers much of the functionality of the Java API as a stateless HTTP/REST service. This service is offered as a web application context (appropriately called service) and is bundled together with a simple search frontend demo application, some additional documentation, and a built-in Jetty HTTP server inside the Document Clustering Server (DCS).

In the examples below we will refer to the DCS and the REST service interchangeably, although the service application context could in theory be deployed on any other web application container (such as Apache Tomcat).

Installation

The DCS is shipped with the Carrot2 distribution under dcs folder. It requires java to be installed on the host system and will bind to port 8080 by default. An alternative port can be selected by passing --port option to the launch script: 

> dcs --port 8081
09:47:32: DCS context initialized [algorithms: [Bisecting K-Means, Lingo, STC], templates: [frontend-default, lingo, stc, bkmeans]]
09:47:32: Service started on port 8081.

Once started, the service is ready to accept requests (typically at http://localhost:8080/service/).
